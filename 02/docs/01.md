# 🎬 项目阶段性总结 — 01

## 概述

为平面设计网站（Next.js + Tailwind CSS + Framer Motion）搭建完整的**国际化（i18n）系统**，包括语言切换、内联字典、动画过渡等。所有文案采用高端显贵行业术语，摒弃平庸机翻。

---

## 步骤详解

### 1. 创建语言 Context (`src/context/LanguageContext.tsx`)

- 基于 React Context + `useState` / `useCallback` 构建轻量国际化系统
- 支持中英文 (`'en' | 'zh'`) 两种语言
- 未使用外部 i18n 库（如 `react-i18next`），全部内联定义
- 从 `localStorage` 存储/读取用户语言偏好（key: `studio-locale`）
- 提供 `useLanguage()` hook 供组件消费
- 解决水合不匹配：SSR 时默认渲染英文，客户端水合后再同步真实语言

### 2. 编写中英文内联字典

所有文案直接定义在 `LanguageContext.tsx` 内部，无额外 JSON 文件。

**字典结构：**
```
header       → 导航栏（STUDIO NOUVEAU / 开启项目 / 服务 / 联系）
hero         → 首屏（标题、副标题、滚动提示、CTA）
portfolio    → 作品集（精选作品、策展式作品集、查看详情）
services     → 服务（定制服务、投资方案、价格切换、咨询）
contact      → 联系（携手共创、表单字段、感谢页、预购定金）
client       → 客户门户（订单、待付款、尾款支付）
footer       → 版权
```

### 3. 重构 LanguageSwitcher (`src/components/LanguageSwitcher.tsx`)

- 右上角 **EN / 中** 按钮
- Framer Motion 上滑/下滑动画：旧文字上滑消失，新文字下滑进入
- 当前选中语言高亮（`accent` 色 + font-weight 500）
- 点击时调用 `toggleLanguage()` + 更新 localStorage
- 修复了用 `try/catch` 包裹 `useLanguage()` 导致 `toggleLanguage` 变成空函数的问题

### 4. 更新 ClientLayout (`src/app/ClientLayout.tsx`)

- 将 `<LanguageProvider>` 包裹整个应用
- 在固定导航栏中渲染 `<LanguageSwitcher>` 和 "START A PROJECT" 链接

### 5. 更新所有组件使用新的 `useLanguage()` 调用方式

**变更：** `t('key.path')` → `t.key.path`

| 组件文件 | 修改内容 |
|---------|---------|
| `Hero.tsx` | 所有 `t('hero.xxx')` 改为 `t.hero.xxx`，`t('header.xxx')` → `t.header.xxx` |
| `Portfolio.tsx` | `t('portfolio.xxx')` → `t.portfolio.xxx` |
| `Services.tsx` | `t('services.xxx')` → `t.services.xxx` |
| `Contact.tsx` | `t('contact.xxx')` → `t.contact.xxx` |
| `Client Portal` | `t('client.xxx')` → `t.client.xxx` |

### 6. 创建 BlurTransition 组件 (`src/components/BlurTransition.tsx`)

- 通用 Wrapper 组件，包裹任意文字元素
- 使用 Framer Motion 实现 `filter: blur(8px) → blur(0px)` + `opacity: 0 → 1`
- 动画时长 **0.3s**，缓动曲线 `[0.25, 0.46, 0.45, 0.94]`（电影感减速曲线）
- 通过 `locale` 作为 React `key` 驱动重渲染
- 支持 `as` prop 渲染为任意 HTML 元素 (`span` / `p` / `h2` 等)

### 7. 应用 BlurTransition 到各组件

| 组件 | 应用位置 |
|------|---------|
| `Hero.tsx` | 顶部标签 `{t.header.studio}`、副标题 `{subtitleStr}`、CTA 文字 `{t.hero.cta}`、滚动提示 `{t.hero.scroll}` |
| `Portfolio.tsx` | 区块标签 `{t.portfolio.tag}`、区块标题 `{t.portfolio.title}` |
| `Services.tsx` | 区块标签 `{t.services.tag}`、区块标题 `{t.services.title}` |
| `Contact.tsx` | 区块标签 `{t.contact.tag}`、区块标题 `{t.contact.title}` |

### 8. 修复 v1/v2 layout hydration 问题

- v1 layout 不再重新声明 `<html>` / `<body>` 标签，仅提供纯内容
- v2 layout 移除嵌套 `<html>` 声明，避免 hydration mismatch 报错

### 9. 修复语言切换按钮点击无效问题（两次修复）

**第一次尝试 — `LanguageSwitcher.tsx`：** 移除 `try/catch` + `let` fallback 变量，直接调用 `useLanguage()`。

**第二次尝试 — 找到真正根因：** `v1/layout.tsx` 和 `v2/layout.tsx` 各包裹了一层独立的 `<LanguageProvider>`，导致嵌套了两个互不影响的 Provider：
- `<LanguageSwitcher>` 在导航栏中，读取**外层** Provider 的状态
- `<Hero>` / `<Portfolio>` 等组件在子 layout 中，读取**内层** Provider 的状态（始终为 `'en'`）

**修复：** 从 `v1/layout.tsx` 和 `v2/layout.tsx` 中移除重复的 `<LanguageProvider>`，确保全局只有一个 Provider 实例。

## 效果演示

1. 打开 `http://localhost:3000`（或 `/v1`、`/v2`）
2. 点击右上角 **EN / 中** 按钮切换语言
3. 所有区块标签和标题文字以 **0.3s blur 滤镜过渡**渐显，营造电影感对焦效果
4. 语言偏好自动保存在浏览器 localStorage 中
